LLM & Agent Core Setup
F-01 LLM platform evaluation & selection Benchmark Claude, GPT-4o, Gemini, and open-source models (Llama 3, Mistral) against PIM-AI tasks. Define selection criteria: tool-calling capability, context window, cost, latency. AI High
F-02 Agent orchestration framework setup Select and configure orchestration layer — LangChain, LlamaIndex, Autogen, or custom. Define agent loop: perceive → plan → act → observe. Set up memory (short-term context + long-term state store). AI High
F-03 Tool / function calling schema design Define the standard JSON schema for all agent tools across the three agents. Includes input/output contracts, error formats, and retry logic. This schema is shared across REST API, DB, and MCP tool calls. Backend High
F-04 MCP server setup & tool registration Stand up MCP server layer. Register tools for catalog read/write, pricing query, inventory check, and order actions. Expose tool manifest for agent discovery. Define permission scopes per agent type. Infra High
F-05 Authentication & agent identity management Implement service account model for agents (OAuth 2.0 / API keys). Define per-agent permission scopes. Audit log all agent actions with identity, timestamp, and payload for traceability. Infra High
F-06 Agent action audit & logging service Build a centralised logging service that records every agent decision, tool call, and outcome. Required for debugging, compliance, and human-in-the-loop review. Store in append-only log with agent ID and session context. Backend High
F-07 Human-in-the-loop (HITL) escalation framework Define confidence thresholds below which agents must escalate to a human. Build notification + approval UI. Agents pause and await approval before executing irreversible actions (e.g. bulk updates, purchase orders). Frontend High
F-08 Shared vector database / semantic search layer Set up vector DB (Pinecone, Weaviate, or pgvector) for semantic product search across all agents. Embed product catalog, descriptions, and attributes. Used by all three agents for natural language product lookup. Infra Medium
F-09 Agent testing & evaluation harness Build automated test suite for agent tasks: unit tests per tool, integration tests per workflow, and LLM eval framework (e.g. LangSmith, RAGAS) for response quality. Include regression tests for prompt changes. AI Medium

Agent 1 — Catalog Management Agent
Autonomously manages product records, classifications, tags, image sequencing, and data quality across the catalog. Acts on behalf of catalog managers and merchandisers.

# Task Description Type Priority
Data Access & API Layer
C-01 Product catalog read API (REST) Build REST endpoints for agent to query products by ID, SKU, category, and attribute filters. Return structured JSON with full attribute schema. Support pagination and bulk fetch. Backend High
C-02 Product catalog write API (REST) Build REST endpoints for agent to create, update, and patch product records. Support partial updates (PATCH). Validate schema on write. Emit change events to downstream systems. Backend High
C-03 Direct DB read access layer Create read-only DB views and stored procedures for agent bulk queries — category stats, data completeness scores, missing attribute reports. Use read replica to avoid impacting production writes. Backend High
Agent Tools (MCP / Function Calls)
C-04 Tool: classify_product Agent tool that takes product attributes + description as input and returns the correct category path (GS1 / eCl@ss / custom). Calls classification model internally. Returns confidence score and top-3 candidates. AI High
C-05 Tool: tag_product Agent tool that analyses product image URL + description and returns suggested attribute tags. Maps to attribute schema. Returns tags with confidence scores. Supports batch mode for multiple SKUs. AI High
C-06 Tool: sequence_images Agent tool that takes a product ID and list of image URLs, identifies image types (hero/lifestyle/detail), and returns the correctly ordered sequence based on category ruleset. Applies marketplace-specific rules. AI High
C-07 Tool: check_data_quality Agent tool that scans a product record or category and returns a data completeness score, list of missing required fields, and recommended actions. Used to drive autonomous enrichment workflows. Backend High
C-08 Tool: bulk_update_products Agent tool for applying updates to multiple products simultaneously (e.g. re-classify entire category, retag a brand's products). Requires HITL approval above a configurable threshold. Returns job ID for async tracking. Backend Medium
Agent Behaviour & Workflows
C-09 System prompt & persona design Design the catalog agent's system prompt: role definition, task scope, tone, escalation rules, and constraints (e.g. never delete records without approval). Include few-shot examples for classification and tagging decisions. AI High
C-10 Autonomous data quality patrol workflow Build scheduled agent workflow that scans the catalog for data gaps, triggers enrichment or tagging tools, and surfaces a daily summary report. Agent runs autonomously but logs all actions for review. AI Medium
C-11 New product onboarding workflow Agent workflow triggered on new product ingestion. Sequentially runs: classify → tag → generate description → sequence images → check quality. Outputs a ready-to-publish product record or flags for human review. AI Medium
C-12 Conflict detection & resolution logic Handle cases where agent-suggested classification or tags conflict with existing human-assigned values. Define resolution policy: agent defers, flags for review, or overwrites based on confidence threshold. Backend Medium
UI & Monitoring
C-13 Catalog agent dashboard Build UI showing agent activity: tasks completed, products processed, quality scores improved, pending approvals, and error log. Catalog managers can review and override any agent action from this view. Frontend Medium
C-14 Agent conversational interface Chat UI allowing catalog managers to instruct the agent in natural language: "Re-classify all products in the Outdoor category", "Show me products with missing descriptions." Agent interprets, plans, and executes. Frontend Medium
C-15 Change history & rollback Every agent write action creates a versioned record. UI allows catalog managers to view before/after for any field and roll back agent changes at product or batch level. Backend Low
C-16 Agent performance metrics & reporting Track agent KPIs: classification accuracy rate, tagging precision, data quality score delta, average time to onboard a product. Export reports for management review. Frontend Low

Autonomously handles catalog queries, product selection, pricing decisions, inventory checks, and purchase order creation on behalf of business procurement teams.

# Task Description Type Priority
Data Access & API Layer
P-01 Catalog query API for agents REST API for structured product search by spec, category, price range, and availability. Optimised for agent consumption — returns ranked results with match scores. Supports natural language query translation to filters. Backend High
P-02 Real-time pricing API Endpoint returning current pricing, discount tiers, contract pricing per buyer, and promotional pricing. Agent uses this to compare options and apply procurement rules (e.g. always select lowest cost within spec). Backend High
P-03 Inventory & availability API Real-time stock levels, lead times, and warehouse locations per SKU. Agent uses to qualify products before recommending or ordering. Triggers substitution logic when stock is insufficient. Backend High
P-04 Purchase order creation API REST endpoint for agent to raise, update, and submit purchase orders. Validates against procurement rules (budget limits, approved supplier list, approval thresholds). Returns PO number and status on success. Backend High
Agent Tools (MCP / Function Calls)
P-05 Tool: search_catalog Agent tool accepting natural language or structured query, returning ranked product matches with full spec comparison. Underpinned by vector search + filter layer. Core tool for all procurement decisions. AI High
P-06 Tool: compare_products Agent tool that takes 2–N product IDs and returns a structured comparison across selected attributes (price, spec, availability, lead time). Used by agent to make or present a recommendation. AI High
P-07 Tool: check_procurement_rules Agent tool that validates a proposed product selection against procurement policy: approved supplier list, budget thresholds, category restrictions, and approval matrix. Returns pass/fail with reason. Backend High
P-08 Tool: raise_purchase_order Agent tool to compile and submit a PO: supplier, line items, quantities, delivery address, and cost centre. Triggers approval workflow if above threshold. Returns PO reference and estimated delivery. Integration High
P-09 Tool: find_substitutes When a product is out of stock or discontinued, agent tool finds equivalent alternatives by spec similarity. Ranks by price, availability, and compliance with procurement rules. Returns substitution rationale. AI Medium
Agent Behaviour & Workflows
P-10 System prompt & procurement rules encoding Design procurement agent system prompt: role, decision authority, escalation rules, and procurement policy encoding. Include constraints: cannot exceed budget without approval, must prefer approved suppliers, must log all decisions. AI High
P-11 Autonomous reorder workflow Scheduled agent workflow monitoring inventory levels. When stock falls below reorder threshold, agent finds best-value restock option, validates against procurement rules, and raises PO automatically (within pre-approved limits). AI Medium
P-12 Procurement request intake workflow Agent accepts procurement requests in natural language (e.g. "I need 200 units of industrial gloves, medium, by end of month, under £8/unit"). Translates to structured search, finds options, and presents recommendation for approval. AI Medium
P-13 ERP / procurement system integration Integrate agent PO output with existing ERP (SAP, Oracle, MS Dynamics) or procurement platforms (Coupa, Jaggaer). Map PIM-AI PO schema to target system fields. Support webhook and API push. Integration Medium
UI & Monitoring
P-14 Procurement agent conversational interface Chat-based UI for procurement users to interact with the agent. Users describe needs; agent searches, compares, and recommends. One-click approval to raise PO from within the interface. Frontend Medium
P-15 Approval & escalation UI Procurement managers review agent-raised POs above threshold. UI shows agent reasoning, product comparison, and procurement rule check result. One-click approve or reject with comment. Frontend Medium
P-16 Spend analytics dashboard Visualise agent-driven procurement activity: spend by category, supplier, cost centre, and time period. Compare agent decisions vs. manual procurement baselines. Track savings identified. Frontend Low
P-17 Agent decision explainability log Every product recommendation and PO raised by the agent includes a structured rationale: why this product, what alternatives were considered, which rules were applied. Stored and viewable by procurement managers. Backend Low

# Task Description Type Priority
Data Access & API Layer
E-01 Product read/write API for content agent REST API for agent to read product records (attributes, existing descriptions, images) and write generated content back. Supports field-level writes — agent can update description without touching other fields. Backend High
E-02 External data sources integration Build connectors to public enrichment sources: Open Food Facts, GS1 registry, manufacturer websites, and open product databases. Agent queries these to fill data gaps. Includes rate limiting, caching, and source attribution. Integration High
E-03 Image ingestion & processing pipeline Build pipeline to accept product images (URL or upload), run quality assessment, pass to enhancement model, and store enhanced versions. Agent triggers this pipeline and receives processed image URLs on completion. Infra High
Agent Tools (MCP / Function Calls)
E-04 Tool: generate_description Agent tool taking product attributes, category, and target channel as input. Returns SEO-optimised title and description. Supports tone/style config per brand. Includes keyword injection from category SEO rules. AI High
E-05 Tool: extract_label_data Agent tool accepting a product image URL. Runs OCR + AI extraction to return structured data: ingredients, nutritional values, allergens, certifications, and claims. Returns with field-level confidence scores. AI High
E-06 Tool: enrich_product Agent tool that takes a product ID, identifies missing required fields, queries external sources and LLM knowledge, and returns enrichment suggestions with source attribution. Agent applies enrichments above confidence threshold. AI High
E-07 Tool: enhance_image Agent tool accepting image URL and returning enhanced version URL. Applies sharpening, background removal, and colour correction via image AI model. Supports batch processing. Returns before/after metadata. AI High
E-08 Tool: translate_content Agent tool for translating generated descriptions and extracted data into target languages. Maintains technical accuracy for spec fields. Supports brand tone guidelines per locale. AI Medium
Agent Behaviour & Workflows
E-09 System prompt & brand voice design Design content agent system prompt: role, content quality standards, SEO rules, brand voice constraints, and escalation criteria. Include few-shot examples of good vs. poor descriptions per category. AI High
E-10 New supplier onboarding workflow Agent workflow triggered when new supplier products are ingested. Reads label images → extracts data → fills gaps via enrichment → generates descriptions → enhances images → outputs publish-ready records. AI Medium
E-11 Scheduled enrichment patrol Agent runs nightly scan for products with low content scores. Prioritises enrichment queue by commercial importance (e.g. top sellers first). Processes batch and reports completeness improvement. AI Medium
E-12 Content approval & review workflow Agent-generated content enters a review queue before publishing. Editors can approve, edit, or reject with feedback. Rejected content with feedback is fed back to improve future generation (RLHF loop). Frontend Medium
UI & Monitoring
E-13 Content agent dashboard UI showing content agent activity: descriptions generated, labels extracted, images enhanced, enrichments applied. Content completeness scores by category. Pending review queue for human editors. Frontend Medium
E-14 Content quality scoring model Build automated scoring model that evaluates generated descriptions against quality criteria: completeness, SEO keyword coverage, readability, and brand tone compliance. Score displayed alongside each generated content piece. AI Low
E-15 Enrichment source traceability Every enriched field stores its source (public DB, API, LLM, label extraction) and extraction date. Displayed in product record for transparency. Supports audit and compliance requirements. Backend Low
Recommended Build Sequence
Phase 1 — Foundation
F-01 to F-07: LLM selection, orchestration, MCP setup, auth, audit logging, HITL framework
Phase 2 — Agent 1
Catalog Management Agent (C-01 to C-16): APIs, tools, workflows, dashboard
Phase 3 — Agent 3
Content & Enrichment Agent (E-01 to E-15): Label extraction, description gen, enrichment
Phase 4 — Agent 2
Procurement Agent (P-01 to P-17): Catalog query, pricing, inventory, PO creation, ERP integration
